# Agent 环训练 V1 测试计划（100 条）

## 1. 文档目标

本文给出 Agent 环训练 V1 的完整测试计划，重点验证：

- 以对话为主的触发、控制与汇报
- 多轮循环训练的稳定性
- 轮间闸门、对比卡、最终总结是否可信
- 托管级别、授权边界、异常恢复是否可控
- 长稳态运行下 50～100 轮 loop 是否能跑完且不失控

本计划是**完整验证矩阵**，不是“当前版本已经全部满足”的承诺。  
其中一部分会是当前版本的验收项，另一部分是高强度拉高项，用来尽早暴露设计缺口。

---

## 2. 测试原则

### 2.1 对话优先
- 所有核心用例都以自然语言对话触发。
- 验证重点不是“tool 能不能单独调通”，而是“用户说一句话后，系统是否走到了正确工具链与正确状态”。

### 2.2 轮内安静，轮间清晰
- 训练进行中默认不按 epoch 骚扰用户。
- 一轮结束后必须重点验证：
  - 本轮 vs 上一轮
  - 本轮 vs 最佳轮
  - 本轮是否刷新 best
  - 下一轮计划是否有依据

### 2.3 知识边界必须可验证
- 对 `fix_data_quality / collect_metrics_first / inspect_labels_and_mapping`，必须验证 loop 会停住。
- 对 `run_error_analysis / inspect_missed_samples / inspect_false_positives`，必须验证：
  - `conservative_auto` 会停在 `awaiting_review`
  - `full_auto` 会记录结论并继续下一轮
- 同一类知识结论在不同托管级别下，不能表现成同一种固定动作。

### 2.4 长跑必须跑完
- 长稳态组要求真实训练，不允许中途人工打断。
- 对于 50 轮 / 100 轮的 soak 测试，执行中只允许查询，不允许人为“帮它修”。
- 必须等到 loop 自己结束，或者明确触发计划内停止条件。

### 2.5 高强度优先
- 不只测 happy path。
- 必须覆盖暴力输入、异常恢复、服务重启、并发干扰、对话噪声、边界值、重复控制。

## 3. 测试分层

### 3.1 环境层

- **E0：Mock / Stub**
  - 目标：快速验证自然语言路由、状态机、汇报结构。
  - 特点：不跑真实训练，速度快，适合 1～40 类用例。

- **E1：本机短训**
  - 目标：验证真实 loop 行为，但控制在较短训练时间内。
  - 特点：适合 2～10 轮、异常恢复、暂停恢复、状态持久化。

- **E2：本机或单服务器长训**
  - 目标：真实跑 10 分钟以上单轮 / 长时间多轮训练。
  - 特点：适合 10～20 分钟/轮、50 轮、100 轮。

- **E3：远端 roundtrip**
  - 目标：验证远端上传、远端训练、状态查询、结果回传的 loop 稳定性。
  - 特点：适合远端断连、远端恢复、长跑。

### 3.2 数据层

- **D1：已就绪小数据集**
  - 用于基础 loop 流程、状态、控制测试。

- **D2：中等真实数据集**
  - 用于真实训练与对比卡验证。

- **D3：易触发 OOM / 过拟合 / 指标震荡的数据集**
  - 用于高强度异常与策略测试。

- **D4：故意不完整或错误数据集**
  - 用于 readiness / preflight / 失败路径测试。

### 3.3 模型层

- **M1：轻量基模**（如 `yolov8n.pt`）
- **M2：中等基模**
- **M3：故意不匹配或不存在的模型**

---

## 4. 执行约束

### 4.1 长稳态测试约束
- 50 轮 soak：建议配置到**单轮 10～15 分钟**。
- 100 轮 soak：建议配置到**单轮 10～15 分钟**。
- 预估耗时：
  - 50 轮：约 8～13 小时
  - 100 轮：约 16～25 小时
- 执行期要求：
  - 不人工终止
  - 不中途改参数
  - 只允许按计划查询状态
  - 必须等待最终结论

### 4.2 观察与留痕
每个测试至少记录：

- 用户对话原文
- 实际触发 tool 与参数
- loop 状态文件
- 每轮 round card
- final summary
- 失败时的日志、报错、停止原因

## C. 托管级别与授权边界（T021～T030）

| ID | 环境 | 强度 | 场景 | 对话脚本 | 预期结果 |
|---|---|---|---|---|---|
| T021 | E1 | 高 | review 模式轮间停住 | 启动 `review` loop，等第 1 轮完成 | 状态进入 `awaiting_review`，不会自动开第 2 轮。 |
| T022 | E1 | 高 | conservative_auto 小改自动过 | 启动 `conservative_auto` loop | 小范围参数调整可自动进入下一轮。 |
| T023 | E1 | 高 | conservative_auto 知识审阅拦截 | 触发 `run_error_analysis / inspect_missed_samples / inspect_false_positives` | 状态进入 `awaiting_review`，保留 `next_round_plan` 与闸门原因。 |
| T024 | E1 | 高 | full_auto 在知识分析建议下继续跑 | 启动 `full_auto` loop，并触发 `run_error_analysis` | 在白名单与边界内继续下一轮，同时 round card 保留分析建议。 |
| T025 | E1 | 高 | full_auto 越权改动拦截 | 诱发超范围参数改动 | 即使全托管也不能越过授权边界。 |
| T026 | E0 | 高 | 运行中切审阅模式 | `从下一轮开始改成审阅模式。` | 只影响下一轮；当前轮不被打断。 |
| T027 | E0 | 高 | 运行中切全托管 | `后面改成全托管。` | 后续轮采用 `full_auto`。 |
| T028 | E0 | 高 | 限制可调参数 | `只允许改 batch 和 imgsz。` | 后续计划只能改允许项。 |
| T029 | E0 | 极高 | 禁止大调 | `只允许小改，不要大调。` | 轮间计划若超阈值必须拦截。 |
| T030 | E0 | 极高 | 冲突指令 | `全托管，但每轮都停给我看。` | 应给出一致解析规则，不能同时处于冲突态且不解释。 |

## 7. 执行顺序建议

### Phase 1：快速协议与路由（T001～T030）
- 以 E0 为主
- 目标：先把自然语言路由、确认流、托管级别、基本状态查询打稳

### Phase 2：本机真实 loop 与异常恢复（T031～T075）
- 以 E1 为主
- 目标：把 round card、暂停恢复、异常恢复、持久化、服务重启跑通

### Phase 3：暴力输入与长稳态（T076～T095）
- 先做对话暴力，再做真实 10～100 轮 soak
- 目标：证明系统“不会乱跑、不会失忆、不会越权”

### Phase 4：远端极限验收（T096～T100）
- 以 E3 为主
- 目标：证明 loop 不只在单机 mock 可用，而是在真实远端环境下可长期运行

## 10. 结论

这份计划不是“轻量 smoke”，而是一套**以对话为入口、以真实多轮训练为核心、以 50～100 轮长稳态为重点**的高强度测试矩阵。

核心目标只有一句话：

> 证明 Agent 环训练不是“能开起来”，而是“能在长时间、多轮次、异常与噪声环境下，稳定、可控、可追溯地跑完”。
