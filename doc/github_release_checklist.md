# GitHub 发布清单

## 当前已完成

- 补充仓库根 `README.md`
- 将公开文档、脚本、测试中的本地绝对路径和私网主机信息替换为公开版示例值
- 将远端 roundtrip 脚本的默认值改为可覆盖的公开示例
- 将本地预测/素材 staging 脚本改成更适合公开仓库的默认方式
- 补充 `.gitignore`，避免本地运行产物混入公开提交

## 发布前仍需确认

- 选择并补充正式许可证（当前仓库未附带开源许可证）
- 复核 `.gitignore` 是否覆盖本地新增大文件或临时目录
- 确认是否需要保留所有内部工程文档
- 推送前再次检查 `git diff --cached`，确保只包含本次发布整理批次

## 推荐发布顺序

1. 本地再跑一轮代表性测试
2. 检查 README 与脚本示例是否符合公开预期
3. 建立发布分支或打 tag
4. 推送到 GitHub
5. 补充 issue / roadmap / release notes
